Outline

This module aims to use techniques and ideas expressed in previous modules, by building on foundations with other things I've picked up along the way, noteably from https://material.io/design/environment/elevation.html#elevation-in-material-design

I can restructure the way I've organized my box-shadows to establish page heirarchy and elevation while also while leaving room to improve past present and future components with lighting & motion.

First things first, we need to define several components for various styles of lists that may contain dropdown list. [aside] I thought dropdowns as nav menus however they can be much more dynamic because they can also be inputs to a larger form

We will also need to create atleast 3 different ways to animate/show the dropdown list in a re-usable way

TaskList

  1. Create Several Types of lists
  2. Create way to show/hide the list
  3. Create ways to indicate the functionality of the list (tickers, warnings, loaders/spinners)
  4. Create show/hide animations (mini-library) for each style of list
  5. Organize the components in a blog format/template
    6. Bonus
  6. Create Stripe style dynamic & pageable dropdown menu with multiple nested pages, compents & states

Pre-Reqs For Completion

  1. All components are customizable and able to be recreated easily in any situation where font-size/container-size/color/stack-order may be subject to change
  2. All components may be created easily with basic [html markup -- scss imported file -- javascript utils to dynamically generate content]
  3. Entire page change themes with a simple settings widget/menu to control color from a scss imported file [main, secondary, accent[1,2,3], text-primary, text-secondary, text-accent, text-passive, box-shadow[1,2,3,4,5]
  4. All images/icons/svg must come from a separate file that is retrieve by the server svgs must be nested within something that obscures the path or converted to png
